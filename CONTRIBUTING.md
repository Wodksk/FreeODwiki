# 贡献指南


**欢迎来到 FreeODwiki！**

本项目是一个开源开放的项目，致力于创建一个社区共建的OD百科。

## 温馨提醒

最重要的提示：如欲参与FreeODwiki的Github仓库编辑，请留意你用于编辑的Github账号绑定的邮箱。合作者可以看到你的这个邮箱，可能造成个人信息意外泄露。而且，编辑敏感信息时请务必使用隐私程度较高的邮箱，这样一来，你泄露的邮箱将难以被追踪者用于破解你的个人隐私。

编辑敏感信息时，请一定要保护好个人隐私：不主动问别人的个人信息、不发布自己或别人的个人信息、做好数据脱敏处理。

## 编写规范

> 没有一个规范就会让各个文档像是用固体胶粘起来的一样，或许阅读的各位可能有体会吧。

### 一般文档规范

规范使用汉字及标点（如使用中文标点）；  
使用大写的OD；  
一个文档的第一行一定是标题，如果有返回则放在标题的下一行；  

目录是文档内的索引，具有告诉读者本文档有何内容的作用。如果目录缺失或者不对，请及时修正；  

标题上下行留空；  
不要留有超过一行的空行；  
一般能用markdown完成的事情就不要用html

### 文件夹的使用规范

对于每一类的文档，都需要用一个文件夹专门存储此类文档文件。除非真的有必要且不会引起歧义，否则不要在该文件夹里面再建文件夹，以减少无意义的嵌套。

~~若无必要，请勿增加文件夹的嵌套。~~

### 报告框架

```markdown
# <你报告的标题>

[◀返回](index.md)

待拟定...

```

建议大家投报告的pr的时候，最好在自己的报告的大标题下面第一行加一个返回箭头，就是`[◀返回](index.md)`(直接复制粘贴就行)。

此外，还可以帮我们把整理的活给干了。就是在`/报告/index.md`中的索引部分的最后面，加上自己的索引` - [你的报告标题](./你的文件名.md)`(另建一行，直接复制粘贴就行)

### 超链接

超链接让文档内部与文档外部形成一个整体，让整个站点变得易于浏览。如果超链接有不对的地方，务必及时修正；如果一些本应有超链接的部分没有，请补上。

为了兼容[FreeODwiki网站](https://freeodwiki.org)使用的mkdocs，用于超链接的文件名和超链接自身不得有英文括号`()`，否则会无法正常显示。

### 文件内容

如果你发现文档的一个指向`/文件/`目录的文件缺失，请将相应的文件补上。
